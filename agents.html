Agentic Pipeline Guide

A light-themed HTML reference for how the agentic parts of this Chrome extension work today, based on the root AGENTS.md.

Two agentic paths Selected-text retry loop and full-page modular pipeline are separate systems.
Live route is smaller The foreground route runs router, intent, optional vision, narrator, and guardrails.
Several agents are dormant Strategist, meaning extractor, and diagram generator exist but are not scheduled.

Current Agentic Paths

Selected Text and Legacy Page Retry Loop

src/utils/renarration.js exports agenticRenarrateText(), a self-refinement wrapper around renarrateText().

  • Enabled by chrome.storage.local.useAgenticPipeline.
  • Used by selected text handling in src/handlers/simple.js.
  • Used by the older screenshot/VLM page renarration in src/handlers/page-renarration.js.
  • Runs up to 3 attempts and accepts the best result above a 3.5 threshold.
  • Evaluator suggestions are appended to later attempts as prompt augmentation.
This path does not run the modular agents in src/agents/.

Full-Page Modular Agent Pipeline

The popup's Renarrate Page button sends run-agentic-pipeline. The background handler calls runPipeline() in src/background/orchestrator.js.

Chrome MV3 DOM clone sidebar Remote Gemini VLM OpenAI or WebLLM text

Modular Pipeline Flow

  1. Create a run context with runId, request data, memory stubs, sectionMap, renarrations, guardrails, and validation.
  2. Ask content.js for extract-and-clone; it extracts text segments, assigns DOM IDs, and opens the split-view clone.
  3. If needed, inject content.js and content.css, then retry extraction.
  4. Run runExtraction() to capture screenshot slices, call Gemini VLM, compress page knowledge, and store lastExtraction.
  5. Load user memory with loadMemory(userId) and reading goal from chrome.storage.sync.
  6. Run pipeline-router and execute its routed agent plan.
  7. Apply context.renarrations back into the cloned iframe through apply-dom-renarration.
  8. Store visualizer state, append pipeline logs, and launch guarded background learning agents.

The browser-visible output is replacement text inside the cloned iframe, not a standalone generated article.

Live Agent Plan

src/agents/agent-0-router.js defines the active route table.

  • full: router, intent analyst, visual cartographer, narrator, guardrails.
  • lite: router, intent analyst, narrator, guardrails.
  • translate: router, intent analyst, narrator, guardrails.
  • annotate: router, intent analyst, narrator.

The router uses request keywords and page metadata first. If rules are inconclusive, it asks the LLM with src/prompts/router-classification.md.

Agent Inventory

Active in Routed Foreground Pipelines

pipeline-router

Reads request and page metadata. Writes pipelineType and agentPlan.

intent-analyst

Creates structured intent with goal, depth, style, language, literacy level, and confidence.

visual-cartographer

Runs in the full route. Enriches DOM sections with VLM-derived roles and visual context.

narrator

Renarrates non-excluded sections in concurrent batches and streams section updates.

guardrails

Sanitizes generated text in place and flags stripped XSS-like content.

Background, On-Demand, or Partially Wired

quality-validator

Scores output after the main route. It can set retry flags, but the orchestrator does not loop yet.

memory-manager

Updates semantic, episodic, and procedural memory after pipeline completion.

predictive-adapter

Serves side panel suggestions through the get-predictions message.

feedback-analyst

Intended for feedback learning, but current feedback logging does not call it correctly.

Present but Not Scheduled

  • content-strategist builds renarrationPlan but is not imported into ALL_AGENTS.
  • meaning-extractor writes meaningMap but is not scheduled.
  • diagram-generator exports disabled = true.

Important Current Gaps

The visualizer says "12-Agent Agentic Pipeline", but the live foreground route usually runs 3 to 5 routed agents, plus extraction and background agents.
  • quality-validator can request retry/replan, but runPipeline() does not implement that loop.
  • narrator does not consume strategist output and hardcodes strategy as rewrite.
  • content-strategist, meaning-extractor, and diagram-generator are code assets, not live stages.
  • Bias and factual guardrail checks are not implemented in the current guardrails agent.
  • VLM is remote Gemini. On-device WebLLM handles text only.

LLM and VLM Interfaces

Use src/utils/llm-dispatch.js for text LLM calls.

await callLLM(
  [{ role: 'user', content: promptText }],
  systemPrompt,
  { tier: 'quality', temperature: 0.3 }
);
  • Do not pass a single { prompt } object to callLLM().
  • Use src/utils/vlm-client.js for Gemini VLM calls.
  • Use prompt Markdown from src/prompts/ through loadPrompt() or cached helpers.

Storage and Research Data

  • chrome.storage.sync: tasks, personas, active selections, reading goal, model settings, semantic memory.
  • chrome.storage.local: API keys, research flags, participant ID, procedural memory, last outputs, visualizer state, pipeline logs.
  • Firestore REST: chat sessions, research logs, feedback events, experiment runs, preference history, user preferences, episodic memory.
Do not add hardcoded secrets. Treat research data and user identifiers as sensitive implementation details.

Guidance for Future Agents

  • Read orchestrator.js, message-handler.js, renarration.js, and the target agent file before editing behavior.
  • Check the router plan before assuming an agent module is live.
  • Keep Chrome MV3 service-worker constraints in mind.
  • Preserve content.js message action contracts.
  • If adding retry/replan, wire it explicitly in runPipeline().
  • If wiring feedback learning, call feedback-analyst with (feedbackEvent, context).
  • Run npm run build for source changes. For docs-only changes, git diff --check is enough.
