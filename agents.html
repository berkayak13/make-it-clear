Current Renarration Flow

The extension now has one OpenAI-backed page flow. It is agentic in the sense that the background worker coordinates extraction, reasoning over page context, saved reading goals, and final rendering, but it no longer uses a multi-agent tree.

Full Page Flow

1. User Action

Popup starts the run

popup.js sends run-page-renarration with the active tab id, URL, and title.

2. Orchestration

Background owns state

src/page-flow/orchestrator.js opens the split panel, sends progress updates, and prevents overlapping page runs.

3. Extraction

Text plus screenshots

extract-page.js requests visible text from content.js, captures page slices, and asks OpenAI for compact JSON knowledge.

4. Renarration

Goal-aware rewrite

renarrate-page.js combines the saved reading goal, active task, active persona, raw page text, and extracted facts.

5. Rendering

Plain text panel

content.js renders the final text with textContent inside the split panel. No generated HTML is inserted.

OpenAI Calls

  • callOpenAIJson() uses the Responses API with structured JSON output for page knowledge.
  • callOpenAIText() uses the Responses API for chat, selected text, and final page renarration.
  • All OpenAI calls set store: false.
  • Build-time model settings live in .env via VITE_OPENAI_*.

Reading Goal Flow

  • The popup or side panel starts a chat session with chatbot-new-session.
  • chatbot-send appends messages and gets an OpenAI reply.
  • chatbot-set-reading-goal extracts a structured goal from the transcript.
  • When the user applies the goal, it is saved to chrome.storage.sync.readingGoal.
  • Final page renarration always includes the saved goal when present.

Selected Text

content.js detects selected text and shows a small trigger. Clicking it sends renarrate-text. The response is shown in the floating overlay and can receive feedback.

Side Panel Extraction

The side panel uses extract-page-knowledge to run extraction without creating the final split-panel rewrite. It renders the latest title, topic, summary, facts, entities, and key terms.

Error Handling

Missing API key, tab access failure, screenshot failure, or OpenAI request failure returns a message to the split panel through update-renarration-progress.

Message Contracts

run-page-renarration Popup entry point for full page extraction, final rewrite, storage, and split-panel rendering.
extract-page-knowledge Side panel entry point for extraction only. Stores and returns lastExtraction.
extract-visible-page-text Content script action that returns visible, deduplicated text plus page title and URL.
show-renarration-panel Content script action that opens the split panel and prepares the loading state.
update-renarration-progress Content script action for progress and error text.
render-renarration-text Content script action that writes final output with textContent.
chatbot-* Chat session, message, and reading-goal actions handled by src/handlers/chatbot.js.

Persistent Data

  • chrome.storage.sync: tasks, personas, active selections, prompt template, reading goal.
  • chrome.storage.local: latest extraction, latest page renarration, current chat session id, research settings.
  • Cloud Firestore helpers are isolated in src/utils/firestore-client.js.

Current Boundaries

  • Active: OpenAI Responses API, split panel, side-panel extraction, chat goal flow, selected-text overlay.
  • Removed: alternate provider paths, local model worker code, DOM clone replacement, and old visual debug pages.
  • Build: Vite only bundles src/background/main.js into build/background-entry.js.
