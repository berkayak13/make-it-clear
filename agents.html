OpenAI Renarration Architecture

The extension uses a small background bundle, a content split panel, and OpenAI Responses API calls for extraction and renarration.

Page Flow

  1. Popup sends run-page-renarration with the active tab metadata.
  2. content.js opens the split panel and returns visible page text.
  3. src/page-flow/extract-page.js captures screenshot slices and asks OpenAI for compact structured page knowledge.
  4. src/page-flow/renarrate-page.js combines saved reading goal, active task, active persona, raw page text, and extracted knowledge.
  5. content.js renders final output as plain text with textContent.

Message Contracts

  • run-page-renarration: full page extraction and final split-panel rendering.
  • extract-page-knowledge: side-panel extraction only.
  • extract-visible-page-text: content-script visible text extraction.
  • show-renarration-panel, update-renarration-progress, render-renarration-text, hide-renarration-panel: split-panel UI.
  • chatbot-new-session, chatbot-send, chatbot-set-reading-goal: chat and goal workflow.

Key Files

  • src/utils/openai-client.js: Responses API helper.
  • src/page-flow/: page extraction, renarration, and orchestration.
  • src/background/message-handler.js: central action dispatch.
  • content.js: selected-text overlay and split panel.
  • sidepanel.js: chat and page knowledge viewer.
